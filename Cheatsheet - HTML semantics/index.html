Cheatsheet Project - HTML Semantics

Good semantics in HTML - and coding in general - provide numerous benefits that positively impact both programmers and users alike.

  • Accessibility
    Good semantics allow browsers to interpret code better, which allows them to better enable access for those with disabilities.
  • Search Engine Optimisation
    It allows search engines to better identify the content of the webpage to allow for more accurate search results.
  • Understandability
    It allows other developers (or future you!) to be able to easily grasp the structure of your code to build on or adapt.

In HTML code we can often observe divitis, which is the excessive use of <div> elements. <div> elements are non-semantic and offer no description of their containing information.

It is best practice to avoid <div> elements as much as possible and instead use more semantic elements in their place.

HTML Semantic Terms


Tag Meaning When to use
<header> Header Usually for navigational lists or introductory content.
<nav> Navigation Encapsulates navigation links, usually within the <header> element.
<main> Main Used to encapsulate the main, dominant content of a webpage.
<section> Section Defines elements of a document of the same theme; i.e chapters.
<article> Article Content that would make sense on its own, i.e articles; blogs; comments; magazines.
<aside> Aside Encompasses additional information that can enhance another element, but isn't required.
<figure> Figure Includes images and their grouped <figcaption>.
<figcaption> Figure Caption Links to the parent <figure> and describes the content.
<audio> Audio Similar to <figure>; encapsulates a section of audio tags.
<footer> Footer Usually contains contact information, copyright, terms of use, a sitemap and links to top of page.
<small> Small Often seen within a <footer> or <aside>, containing copyright information or legal disclaimers.
<time> Time Allows an unambigous ISO 8601 date to be attached to a human-readable version of that date.

Additional Semantic Tips

Good use of elements can go a long way, but its also important to practice proper indentation of your code.
This both helps with readability, and also helps a great deal with syling the webpage through CSS.

You can also make your code more readable by including comments to signpost readers, or starting a new line each time a <br> is used.

Inspect this code and see what has been done to improve its semantics and readability!
